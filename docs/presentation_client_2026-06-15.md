| 현장에서 바로 하면 | 시뮬레이션으로 먼저 하면 |

| 어떤 파지 방향이 현실적인지 모름 | 물리(질량·마찰·접촉)로 성공률 측정 |

## 슬라이드 5 — 핵심기술 ① AI 파지 추론 (GraspGen)

**제목:** 제품 위치가 매번 달라도 — AI가 잡을 곳을 판단 (사양 R4·R5)

### 동작 원리

```
손목 카메라 → 제품 3D 점구름(PointCloud)
        ↓
   GraspGen(AI 추론) → 잡을 수 있는 후보 자세 다수 + 신뢰도 점수
        ↓
   현재 장면에 맞는 최적 후보 선택 (다음 슬라이드 cuRobo와 연계)
```

- **고정 좌표가 아니라** 매 순간 제품을 보고 판단 → 제품이 흐트러져 있어도 대응
- 백본: PointNet++ (현 GPU 환경 RTX 5080/CUDA 12.8 호환 확정)
- 추론 서버 분리(ZMQ) → 로봇 제어 프로세스와 독립, 안정성 확보
- 물체 3종(스낵백/캔/병) 대응. 축대칭 캔은 옆파지(side-grasp) 전략으로 안정화

**이미지:** → shot_001_grasps.png (제품 위 여러 파지 후보 좌표축, 선택된 후보 강조)

## 슬라이드 6 — 핵심기술 ② 충돌회피 모션 (cuRobo)

**제목:** 제품·매대·작업자와 부딪히지 않는다 (사양 R3)

### 동작 원리

- 로봇 전체를 **충돌 구체 모델**로 표현, 작업대·매대·이웃 제품을 실시간 장애물로 등록
- 목표 파지 자세 → **충돌 없는 관절 궤적**을 자동 생성 (cuRobo plan_single / plan_grasp)
- 후보 파지를 한꺼번에 넘기면 **플래너가 현재 장면에서 도달 가능한 안전 후보를 자동 선택**

### 고객 가치로 환산하면

| 상황 | 시스템 대응 |
|------|------------|
| 잡으려는 캔 옆에 다른 캔이 있음 | 충돌 없는 반대 방향 접근을 자동 선택 |
| 양옆이 다 막힘 | 정면 접근 후보를 자동 선택 |
| 운반 중 매대 간섭 위험 | 관절 자세를 제한해 안정 경로 유지 |

**이미지:** → shot_004_grasp_reached.png (이웃 제품 사이로 진입해 파지 도달)

## 슬라이드 11 — 정리 (도입 기대효과)

**제목:** 정리 — 무엇을 얻으시는가

- **무인 매대 정리** — 반복 진열 작업 자동화
- **안전 우선 설계** — 충돌회피 + 사전 시뮬 검증으로 장비·인력 리스크 최소화
- **가변 환경 대응** — 고정 좌표가 아닌 AI 파지로 흐트러진 제품도 처리
