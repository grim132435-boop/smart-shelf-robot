# 내부 분석 — 개선·수정 필요사항 (고객 비공개)

> 발표 자료(presentation_client_2026-06-15.md)는 한계를 "고도화 로드맵"으로 긍정 프레이밍한다.

## 0. 한 줄 진단

기능(파지→운반→적치)과 안전(무충돌)은 **실기 사양 동일 디지털 트윈에서 검증 완료**.
그러나 ① 실 하드웨어 미배치, ② 물체 3종 중 캔만 검증, ③ 비전 실연동 부재, ④ 코드 모듈화 미완 — 이 4가지가 현장 도입 전 핵심 갭이다.

---

## 1. 우선순위 P0 — 현장 도입 차단 요소

### 1-1. 실 하드웨어 미배치 (Sim-to-Real 갭)
- **현황:** 모든 검증이 Isaac Sim 내 실기체 USD 모델 한정. 실제 E0509/RH-P12 물리 로봇 구동·검증 이력 없음.
- **리스크:** 시뮬 드라이브 게인·마찰·물리 파라미터가 실측과 다르면 실기 동작 재현 안 됨. 특히 RH-P12 그리퍼 전류↔그립력 관계는 시뮬 모사값.
- **수정 필요:** 실기 1대로 게인/마찰/그립력 캘리브레이션 → 시뮬 파라미터 역보정 루프.

### 1-2. 물체 3종 중 캔만 검증
- **현황:** 사양 R2는 스낵백/캔/병 3종. 실제 픽앤플레이스 클린 런 검증은 **캔 중심**. 스낵백(변형체)·페트병(투명/경량) 미검증.
- **리스크:** 스낵백은 비강체라 GraspGen·물리 파지 거동이 캔과 전혀 다름. 페트병은 점구름 품질(투명)·무게중심 이슈.
- **수정 필요:** 물체별 파지 전략 분기 + 각 물체 단독·혼합 클러터 검증. (configs_draft에 bowl/box/cylinder npz만 존재 → 실제 3종 데이터셋 부재)

### 1-3. 비전 실연동 부재 (perception 갭)
- **현황:** 점구름이 시뮬 물체에서 직접 샘플링됨(sim_source). 실 카메라(D455)→SAM→점구름 파이프라인 미통합. 비전팀 인터페이스(`point_cloud` key, robot base frame) **합의 대기 중**.
- **리스크:** 실 점구름은 노이즈·부분관측·좌표계 오차 포함. 현재는 완전·정확한 시뮬 점구름 가정.
- **수정 필요:** ros2_source 구현 + 비전팀 인터페이스 확정 + 노이즈/부분관측 강건성 검증.

## 2. 우선순위 P1 — 품질·정밀도 결함 (검증되었으나 미흡)

### 2-1. 추종 정밀도 / 중력 처짐
- **현황:** 순수 PD 제어 → joint_2 중력 steady-state 처짐. Stage5에서 중력보상 피드포워드로 4.55°→0.17° 개선 진행 중이나 전 관절·전 구간 적용·검증은 미완.
- **리스크:** 처짐이 매대 진입 시 제품·선반 미세 간섭으로 이어질 수 있음.

### 2-2. 그리퍼 TCP 오프셋 임시값
- **현황:** TCP 깊이가 임시값(약 110mm) 사용 → 캔 직립 후 미세 기울기(~2°). 실 그리퍼 기어박스 오프셋 미보정.
- **리스크:** 적치 정밀도(직립도) 저하. 사양 R5(손상 없는 안정 파지)와 직결.

### 2-3. 캔 파지 방위 = 휴리스틱 합성
- **현황:** 축대칭 캔은 GraspGen 출력을 직접 쓰지 않고 "로봇 정면±각도 azimuth 스윕 + X축 위로 강제"로 side 파지를 **합성**. 일반 물체로 확장 시 이 휴리스틱이 깨질 수 있음.
- **리스크:** 물체 종류 확장 시 물체별 합성 규칙이 늘어나 유지보수 부담 → 모델 일관성 저하.

## 3. 우선순위 P2 — 엔지니어링/구조 부채

### 3-1. 코드 모듈화 미완
- **현황:** ARCHITECTURE.md가 제안한 공용 코어/어댑터 분리 구조 미적용. 단일 stage4/5/6/7 파일에 인식·선택·플래닝·실행·시각화 책임 혼재. 스테이지별로 파일이 분기·복제됨.
- **리스크:** sim→real 이관 시 코어/어댑터 경계가 없어 "한 파일만 교체" 이관 전략이 현재는 불가능. 버그 수정이 여러 파일로 번짐.
- **수정 필요:** transforms → grasp_inference → motion_planning → perception_bridge → robot_adapter 순 점진 분리.

### 3-2. 매대·웨이포인트 좌표 하드코딩
- **현황:** 매대 3단 좌표·배치 웨이포인트가 코드/메모에 고정값. 매대 규격·로봇 설치 위치 변경 시 수작업 재설정.
- **수정 필요:** 매대·작업셀 좌표를 설정 파일/캘리브레이션으로 외부화.

### 3-3. 자동 테스트·재현성 부재
- **현황:** 검증이 PNG 수동 확인 중심. 자동 회귀 테스트 없음. (CLAUDE.md 규칙 8 "완료 전 테스트" 대비 갭)
- **리스크:** 코드 수정 시 회귀 검출 불가. 클린 런 재현이 수동·환경 의존.
- **수정 필요:** 파지 성공/무충돌/사이클 결과를 판정하는 헤드리스 회귀 시나리오.

## 4. 우선순위 P3 — 사양 충족 입증 공백 (고객 질문 대비)

| 항목 | 현황 | 비고 |
|------|------|------|
| 사이클 타임(처리량) | 미측정·미보고 | 고객 ROI 핵심 지표일 가능성 높음 — 우선 측정 권장 |
| 파지 성공률(통계) | 정성적('성공') | N회 반복 통계·실패율 미집계 |
| 안전 인증/E-stop/협동안전 | 미고려 | 작업자 협동 시 속도·힘 제한(ISO/TS 15066류) 미반영 |
| 도메인 강건성(Stage7) | 미완 | 조명/텍스처/위치 랜덤화 일반화 미검증 |
| 적치 실패 시 복구 | 미설계 | 파지 실패·낙하 시 재시도/안전정지 로직 부재 |

## 5. 권장 조치 순서 (현장 도입 관점)

1. **사이클 타임·파지 성공률 정량 측정** (P3, 저비용·고임팩트 — 고객 설득력).
2. **물체 3종 전체 + 혼합 클러터 검증** (P0 R2 충족).
3. **비전 인터페이스 확정 + ros2_source 구현** (P0 — 비전팀 의존, 리드타임 김).
4. **코드 모듈화(코어/어댑터)** (P2 — 실기 이관 전제조건).
5. **실기 1대 캘리브레이션 → 시뮬 역보정** (P0 Sim-to-Real).
6. 정밀도(중력보상 전구간)·그리퍼 오프셋 실측 (P1).
7. 안전 로직(E-stop/복구/협동안전) 설계 (P3 — 실배치 필수).
