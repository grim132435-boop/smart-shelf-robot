## A. 확정 정책 (사용자 2026-06-12)

1. **물체 3종**: 캔(can) / 과자봉지(snack_bag) / 페트병음료(pet_bottle).
2. **층 배정**: 캔 → **2층 매대**, 페트병 + 과자봉지 → **3층 매대**.
3. **시작 순서**: 캔 → 2층 넣기부터 먼저 구현·검증.
4. **타겟 선택 정책 변경**: (구) 매대측(+y) 가까운 것 우선 → **(신) 홈에서 가장 바깥쪽 물체부터**.
   근거(사용자): 바깥쪽 먼저 치우면 후속 파지가 남은 물체 위를 안 지나 충돌 회피·모션 효율↑.
   ※ 이전 노트의 "효율 기반 선택"도 이 "바깥쪽 우선"으로 구체화/대체. feasibility 필터는 유지.
5. **파지점 높이가 핵심**: 캔을 조금 높게 잡으면 매대(특히 2층 천장)와 간섭. 로봇팔+비전 단에서
   파지점 정밀도가 성패를 가름.

## B. ★먼저 풀어야 할 충돌 (BLOCKER — 착수 전 사용자 확인 필요)

**캔 → 2층**은 과거 검증 결과와 정면 충돌한다. 메모리 [[stage4-shelf-coords]] 기록:

> 2단(Shelf1_2): 바닥 world z≈0.95, **천장 있어 개구 ~0.16m → 캔 13.5cm + 여유로 옆파지 그리퍼
> 진입 불가 → 3단(맨 위)로 배치 확정.**

즉 13.5cm 캔을 천장 16cm 칸에 직립으로 넣으면 캔 위 여유가 ~2.5cm뿐 → RH-P12 그리퍼(손목+손가락)가
캔 위에 얹혀 들어가는 옆파지로는 진입·릴리즈가 불가능했다. 사용자의 "조금 높게 잡으면 간섭" 관찰과 동일.

**→ 그래서 Stage7 0순위 = "캔 2층 진입 feasibility 스파이크"(Phase0식 de-risk).** 다음 중 하나로 풀려야 진행:
  - (a) 2층 실제 개구를 --inspect-shelf로 재측정 (USD가 바뀌었거나 측정이 보수적이었을 수 있음).
  - (b) 캔 실제 높이 확정 (실제 음료캔 12.2cm? 13.5cm? 짧으면 여유 생김).
  - (c) **파지점을 더 낮게** 잡아 그리퍼가 캔 위로 덜 솟게 (단 책상 파지 시 INVALID_START_WORLD_COLLISION
        재발 위험 — pick(책상)↔place(천장) 트레이드오프를 수치로 잡아야 함). ← 4번 항목과 직결.
  - (d) 안 되면: 캔도 3층, 다른 배정으로 사용자와 재협의.

## ★ 2층 스파이크 결과 (2026-06-12, --shelf-level 2)
세운 캔 2층 진입 가능성 실측. min_world_clearance(매대 메시 기준)가 권위값.
| 조건 | 최소 간격(하강) | 판정 |
|---|---|---|
| 0.7·half 파지(상단, 기존) | slot_feasible −12mm | 불가(사전 차단 HOLD) |
| 중앙 파지(grasp-frac 0) + ENTRY_CLR 0.005, **카메라 미모델** | +6.7mm | 통과(P8) |
| 위 + RealSense **추정** 충돌구체(내가 0.06 돌출·과대) | +0.3mm | 그레이징(추정 오류) |
| 위 + RealSense **실측** 충돌구체(사용자 Lula 저작, x0.047·r0.02·폭10cm) | **+24.4mm** | 통과(P8, 직립 z1.018) |
- 결론: 사용자 제안(중앙 파지) + RealSense 실측 충돌구체 → **+24.4mm로 통과**(세운 캔 2층 적치 가능).
  ★내 추정 카메라(+0.3mm)는 위치·크기 과대(0.06 돌출, 폭 y가정)였고, 사용자가 Lula 에디터로 실제 카메라
  메시 위에 저작(x0.047·z0.027·r0.02·폭10cm, vision_collision.yaml)하니 실제 여유가 훨씬 큼.
- ★주의(실기체): 간격은 plan_grasp(goalset)이 매 런 다른 grasp/azimuth를 골라 런마다 달라짐. 이번 런 +24.4mm는
  단발 — 실기체 비전오차 마진 위해 worst-case를 여러 런/위치로 바운드해야 안전(joint_6 롤은 그때도 부족하면 적용).
- RealSense 충돌구체(사용자 저작) e0509_spheres.yml의 gripper_rh_p12_rn_base에 병합 완료(추정 5구체 교체).
- **worst-case 바운딩(2026-06-15)**: 세운 캔 dy=-0.1/0/+0.1 모두 하강 간격 +24.4~24.6mm로 **일관**
  (적치 슬롯 고정 → 배치 마진은 픽 위치와 무관). 실기체 비전오차 감안해도 안전. **세운 캔 2층 확정 가능.**
- **누운 캔 = 현재 미지원(중요)**: `--can-pose lying`로 눕혀 spawn하면 IDLE의 "넘어짐 감지→직립 재배치"
  가드가 **시뮬에서 캔을 세워버림**(텔레포트) → 그 뒤 일반 캔처럼 +24.5mm P8. 즉 누운 채 잡는 게 아니라
  리셋. **실기체에선 이 리셋이 불가** → 누운 캔 픽은 별도 기능 필요(수평 실린더 파지 + 직립 재배향).
- 누운 캔은 천장 여유 +100mm로 여유 → 2층은 **누운 캔이 안전한 해법**.
- 추가/보정 후보: 카메라 마운트 각도/위치 조정, 파지 더 하향(책상·그립안정 트레이드오프), 진입 전략 변경.
- 신규 코드(stage7): `--shelf-level{2,3}`(층 라우팅+천장상수), `--grasp-frac`(파지높이배율),
  highest_sphere_top_world/log_ceil_headroom(캔윗면-천장), 2층 ENTRY_CLR 0.005, RealSense 5구체(e0509_spheres.yml).

## D. 추가·개선 제안

1. **그리퍼 전류 기반 파지 확인** (비전 역할 축소 반영, 노트 2026-06-12): 파지 성공/유지(들림·접촉)는
   비전이 아니라 그리퍼 전류(effort/관절 추종)로 판정. 현재 "리프트 후 물체 z" 체크를 전류 신호로 보강.
2. **층별 슬롯맵 분리**: 현재 SHELF3_SLOTS 단일 → 2층/3층 각각 점유맵. 페트병/과자 2종이 3층 공유.
3. **과자봉지 파지 자세**: 봉지는 변형체(deformable)라 강체 근사 한계. 우선 강체 box로 top 파지,
   변형 모델은 범위 밖(사용자 합의 필요).
4. **도메인 랜덤화(원래 Stage7 목표)는 후순위**: 위 다물체 라우팅이 선 완료된 뒤. perception이 Mock이라
   조명/텍스처 랜덤화는 비전팀 합류 전엔 효과 제한적(노트 기록).
5. **타입별 GraspGen 체크포인트**: 현재 robotiq_2f_140 단일. 물체 형상이 다양해지면 side/top 모드 매핑과
   합성 변환(robotiq→RH-P12)이 타입마다 맞는지 재확인.
6. **모션 연속화(택타임 최적화) — 사용자 관찰 2026-06-15**: 현재 파지→리프트(z+)→매대앞 이동→매대 안 진입이
   세그먼트별 plan_single/moveL로 끊겨 정지감이 있음. 택타임 잡을 때 **하나의 연속 궤적으로 블렌딩**해야 함.
   (Stage5서 moveL settle=0으로 핸드오프 일부 연속화했으나, grasp→lift→carry→insert 전체 체인은 여전히 분절.)
   접근: 웨이포인트 통합 trajopt(중간정지 없는 단일 plan) 또는 세그먼트 경계 속도 연속(blend radius)·정착대기 제거.
   ★우선순위는 정확도/안전 확정 후. 지금은 단계별 검증 가능성이 더 중요해 분절 유지.
7. **누운 캔 픽(실기체 필수)**: 현재 누운 캔은 "넘어짐→직립 재배치" 가드가 시뮬서 텔레포트로 세움(실기체 불가).
   실제 누운 캔 픽 = 수평 실린더 파지(top 또는 축방향 side) + 그리퍼 재배향으로 직립화 후 적치. 별도 기능 필요.
   → (2026-06-15 구현) lying_grasp_from_axis(위에서 X=캔축 파지→carry가 직립화), can_is_lying 타겟별 자동감지.
8. **물체 물성 정밀화(실기체 필수) — 사용자 2026-06-15**: 현재 모든 물체 질량 0.30·마찰 1.5 동일(근사).
   실제로는 **물체별 정확한 질량·마찰계수** 적용 필요(캔 빈/full, 페트병 내용물, 과자봉지 경량).
   누운 실린더는 굴러 이탈 → 임시로 angular/linear damping(50/5) 부여했으나, 실물 물성으로 대체해야 함.
9. **과자봉지 변형(deformable) — 사용자 2026-06-15**: 봉지는 파지 시 변형되는 연성체. 강체 박스 근사로는
   접촉·파지 거동이 부정확. IsaacLab DeformableObject로 변형 모델 적용 필요(파지력·접촉면 재현). 범위 큼.

## 과자봉지 변형체 — ✅ FEM volume 성공 (2026-06-15)
- **방향전환 확정**: particle cloth inflatable(평평→pressure 부풀림)은 cm 스케일서 불안정(풍선 진동,
  p60 63→27mm). 봉지는 늘어나는 풍선이 아니라 포일이라 부적합. → **FEM volume deformable(soft body)** 채택.
- **작동 레시피(deformable_bag_spike.py)**: 닫힌 박스 0.16×0.23×**0.07**(실측 부푼 형태) → add_physx_deformable_body
  (hexres 10, collision_simplification) + add_deformable_body_material(**youngs 0.3MPa**, poisson 0.45,
  dyn_friction 0.6, elasticity_damping 0.01, density 120) + ★**GPU dynamics 활성화**(PhysxSceneAPI EnableGPUDynamics).
- **결과**: rest 70mm 안정 유지(붕괴·진동 없음), presser로 누르면 **납작하게 찌그러지며 옆 부풀음**(스낵봉지 느낌,
  fem_150.png). Young's로 물렁함 조절(2MPa=단단/안 눌림, 0.3MPa=잘 눌림). Blender 메시 불요(FEM 자동 사면체화).
- 측정 주의: max-min z는 비압축성이라 국소 찌그러짐 못 잡음 → 샷으로 확인.
- ★남은 통합: GraspGen엔 봉지 표면 PC, cuRobo 충돌은 bbox/convex 근사(deformable 직접 장애물 불가), 파지+3층 적치.
