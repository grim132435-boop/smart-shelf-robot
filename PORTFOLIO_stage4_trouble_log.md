# 매대 정리 픽앤플레이스 — GraspGen→cuRobo→Isaac Sim 트러블슈팅 정리
**(Isaac Sim / 로봇 매니퓰레이션 직무 포트폴리오용)**

> 프로젝트: E0509 6축 로봇 + RH-P12-RN-A 그리퍼로 책상 위 캔을 파지해 매대에 정리.
> 파이프라인: **GraspGen(파지 자세 추론) → cuRobo(충돌회피 모션플래닝) → Isaac Sim(물리검증)**.
> 환경: RTX 5080(Blackwell)/CUDA 12.8/torch cu128, Isaac Sim 5.1, cuRobo v0.7.x(v1 API).

---

## STAGE 1 — GraspGen 파지 추론
**목표:** 물체 점구름 → 파지 자세(M,4,4)+score.

| 문제 | 원인 | 해결 |
|---|---|---|
| PTV3 백본이 Blackwell/cu128에서 빌드 실패 | PointTransformerV3 커스텀 CUDA 커널이 환경 비호환 | **PointNet++ 백본으로 고정**, GraspGen을 ZMQ 추론 서버로 분리해 메인 프로세스와 디커플 |
| 대칭 원통캔에서 GraspGen 파지 방위가 불안정/난잡 | 학습 분포가 일반 물체 위주, 축대칭 물체는 다중해 | 캔은 GraspGen 방위를 그대로 안 쓰고 **로봇 정면±각도 azimuth 스윕으로 side 파지 재합성**(아래 Stage3) |

**역량:** 딥러닝 추론 파이프라인 환경 트러블슈팅, 프로세스 분리(IPC) 설계.

---

## STAGE 2 — cuRobo 모션플래닝
**목표:** 목표 pose → 충돌 없는 궤적.

| 문제 | 원인 | 해결 |
|---|---|---|
| EE가 목표를 한참 벗어난 공중(z≈1.48)으로 감 | cuRobo는 **base_link 원점 기준**으로 계획하는데 로봇이 월드 [-0.25,-0.04,0.73]에 있어 월드 타겟을 그대로 주면 +base만큼 어긋남 | 월드↔base 변환에 `_ROBOT_BASE_OFFSET` 차감. **FK(관절)+base == 실제 EE 월드(검증 차이=0)** 로 확정 |
| 책상 위 모든 파지를 충돌로 거부(거짓 데드존) | `collision_table.yml` 슬랩(5×5×0.2)이 base프레임 기준이라 기본값이면 윗면이 로봇 마운트(0.73) 바로 아래에 깔림 | 슬랩을 실제 바닥(월드0)으로 내림: `pose[2]=-base_z-dims_z/2` |
| V2 평가용 API 혼선 | v0.7.x엔 V2 API(MotionPlanner.plan_pose) 없음 | **v1 API(MotionGen.plan_single, IKSolver) 고정** |

**역량:** 좌표계(프레임) 디버깅, FK 검증, cuRobo 내부 충돌모델 이해.

## STAGE 3 — GraspGen→cuRobo 연결 + 파지 디버깅 (가장 난이도 높았던 구간)

### 3-1. 충돌구체(collision sphere) 모델이 근본적으로 틀려 있었음 ⭐
**증상:** 로봇이 책상에 부딪히는데 cuRobo는 "충돌 없음"으로 통과.

- **원인:** 기존 `e0509_spheres.yml`의 구체가 전부 링크 **로컬 z축(x=0,y=0)** 에만 있었음.
  실제 메시를 트리메시로 로드해 bbox를 찍어보니 **link_2는 X축으로 0.49m 긴 링크**인데
  구체가 z축에만 있어 통째로 비어 있었음 → cuRobo가 이 링크를 충돌검사에서 못 봄.
  (시각화로 확인: 구체가 팔에서 옆으로 떠 있었음)
- **해결:** cuRobo **공식 `fit_spheres_to_mesh`**(`curobo.geom.sphere_fit`, VOXEL_VOLUME)로
  각 링크 메시를 복셀화해 재생성. 팔=실제메시/convex-hull 복셀, 그리퍼=표면샘플(손가락 사이 안 채움).
  base_link은 책상 마운트라 항상 겹쳐 `INVALID_START` → `collision_link_names`에서도 제외.
- **검증:** grasp 자세에서 **책상 아래 구체 0개** = 팔이 책상 위 유지. 거짓 충돌/실제 충돌 모두 정확해짐.

**역량:** 충돌모델 정확도가 모션플래닝 신뢰성의 근간임을 규명. 메시 처리(trimesh), cuRobo 구체 피팅 API 활용, 시각화 기반 디버깅.

### 3-2. RH-P12 그리퍼 side 파지 합성
- **문제:** GraspGen은 robotiq(닫힘축 X) 기준, RH-P12는 **base Y축으로 닫힘**(URDF상 손가락 ±y, x축 회전). 직접 변환 시 자세 틀어짐.
- **해결:** GraspGen의 approach 방위만 취해 **RH-P12용 파지를 합성**(`side_grasp_from_approach`):
  Z=approach(수평), Y=수평접선(손가락분리), **X=위로 강제(그리퍼 위 RSD455 카메라가 하늘 향함→책상충돌 회피)**, EE=캔중심−TCP깊이·approach.

### 3-3. 드라이브 게인 미적용(팔 출렁/처짐)
- **원인:** IsaacLab export USD의 관절 드라이브 게인이 **placeholder(kp=100,kd=1)**. (IsaacLab은 게인을 런타임 파이썬에서 주입하는 구조라 USD 단독 로드 시 미적용.)
- **해결:** 실기 모사 검증값 주입 — arm kp800/kd80/eff200, gripper kp2000/kd100/eff50. 텔레포트 후 드라이브 타겟도 home으로 설정(안 그러면 PD가 0으로 끌어 처짐). 순수 PD라 중력보상 없음→joint_2 ~2° steady-state 처짐 인지.

### 3-4. 물리 파지(마찰로 캔 들기) — 다단계 디버깅 ⭐⭐
**증상:** 그리퍼가 닫혀도 캔이 안 들리거나, "그리퍼가 캔을 통과한 뒤 캔이 날아감".

순차적으로 원인을 분리해 해결:
1. **grasp 높이가 너무 높음**(0.8·half=캔 윗면) → 미끄러짐 → 몸통(0.3·half)으로 낮춤.
2. **그리퍼가 접근 전 안 열려 있음** → QUERY_GRASP에서 `set_gripper(OPEN)` 추가.
3. **슬램 닫기**(stiffness 2000, 목표 한번에) → 캔을 쳐냄 → **현재각→목표 램프 보간**.
4. **TCP 깊이 오류**: 0.1034(robotiq값)는 캔을 손가락 끝에 놓아 닫을 때 캐밍으로 빠짐.
   손가락 메시 치수 측정(끝 z≈0.11, 케이지 중심 0.05~0.06) → **0.060으로 깊이 보정**(케이지에 seat).
5. **캔 안착 높이 들쭉날쭉(0.73/0.75)**: 재배치 높이 _cz가 책상면보다 0.035 높아 떨어지며 튀고 기움
   → `_cz=table_top+half+2mm`, **재배치 시 orientation=[1,0,0,0]+속도0 리셋**, settle 대기 후 위치 재측정.
6. **★최대 돌파구 — 컴플라이언트 그리퍼:** 강성이 높으면 닫을 때 캔을 **들이받아 관통/발사**(PhysX 접촉 미해결).
   **gripper kp 2000→600, effort 50→25**로 낮추니 손가락이 캔에 부드럽게 멈춰 물림(r1~0.89). → **물리 파지 성공률 ~50%→사실상 100%(연속 6/6)**.
7. **신뢰성 보강:** 리프트 실패 시 **재파지(re-grasp) 최대 3회**, 도달성 실패 시 HALT 대신 재배치 후 재시도.
8. **리프트:** grasp가 책상 근처라 plan_single이 start-collision으로 거부 → **직접 IK+관절 보간 수직 이동**(그리퍼 닫힘 유지, 캔 마찰로 딸려 올라옴).

**역량:** PhysX 접촉/드라이브 동역학 이해(강성-관통-발사 관계), 적응형 그리퍼 케이징 기하, 마찰/머티리얼, 물리 파지 안정화. 가설→측정→격리 방식의 체계적 디버깅.

### 3-5. 운영 자동화 (디버깅 효율)

## 핵심 회고 (포트폴리오 어필 포인트)
1. **"충돌모델 정확도가 모션플래닝 신뢰성의 전제"** — 구체가 링크와 1cm만 어긋나도 cuRobo가
   실제 충돌을 못 보거나 멀쩡한 자세를 거부함. 시각화·FK검증으로 근본 원인 규명.
2. **물리 파지는 단일 원인이 아니라 기하(깊이)·동역학(강성)·접촉(마찰)·초기조건(안착)의 합** —
   하나씩 격리해 해결, 특히 **컴플라이언트 제어가 관통/발사를 푸는 열쇠**였음.
3. **sim 충실도와 도달성/안정성의 트레이드오프**를 정량적으로 다룸(게인, 구체 밀도, grasp 높이).
4. **헤드리스 디버깅 파이프라인 구축**(자동 PNG, 진단 로그, 세션 자동탐지)으로 반복 검증 효율화.
